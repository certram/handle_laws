# 开发需求

现在有新的功能。就是生成的文件最终命名协助执行通知书，改为中文名.docx，如下，命名只取双引号内的内容

1. 房产类，“协助执行通知书（查封xx房产）.docx”。xx指的是房产所在地。举例子，假设深圳市的房产内多套房产，应该做如下逻辑处理：协助执行通知书（查封深圳房产一）.docx，协助执行通知书（查封深圳房产二）.docx。
2. 银行类的，“协助执行通知书（银行xxxx）.docx”。xxxx是四位数字，你需要从对应的银行账号截取尾号4位阿拉伯数字，以此避免同名。如果有同名的，比如当银行账号尾号4相同时，会发生重名，处理逻辑：在其后面协助执行通知书（银行xxxx）-1.docx，协助执行通知书（银行xxxx）-2.docx，以此类推做处理。
3. 车辆类，“协助执行通知书（查封xx车辆）.docx”。xx指的是车辆所在地。举例子，假设是深圳市内有多辆车辆，应该做如下逻辑处理：协助执行通知书（查封深圳车辆一）.docx，协助执行通知书（查封深圳车辆二）.docx。
4. 支付宝，“协助执行通知书（支付宝）.docx”，如果有多个支付宝账号，协助冻结存款通知书（支付宝一）.docx，
协助冻结存款通知书（支付宝二）.docx，以此类推做处理。
5. 财付通，“协助执行通知书（财付通）.docx”，如果有多个支付宝账号，协助冻结存款通知书（财付通一）.docx，
协助冻结存款通知书（财付通二）.docx，以此类推做处理。
6. 股权类。协助执行通知书（冻结股权）.docx。如果有多个股权，协助冻结存款通知书（冻结股权一）.docx，
协助冻结存款通知书（冻结股权二）.docx，以此类推做处理。

## 新需求一

1. 保全卷，具体命名逻辑："xx民初"+xxxx+"保全卷".docx。“xx民初”，在settings.yaml配置一个字段civil_first，值配置为：“26民初”，一般不常更改。xxxx这里是：从案件的文件名截取，例如案件名：（2026）粤0305民初7226号-20260323，截取“民初”后面到“号”前面的数字，用正则匹配即可。具体例子：26民初13179保全卷.docx。

2. 裁定书，具体命名逻辑："xx民初"+xxxx+"保全裁定书（xxxxxx）".docx。字符串拼接。具体一个例子，26民初13179保全裁定书（260416）.docx。“xx民初”，在settings.yaml配置一个字段civil_first，值配置为：“26民初”，一般不常更改。xxxx这里是：从案件的文件名截取，例如案件名：（2026）粤0305民初7226号-20260323，截取“民初”后面到“号”前面的数字，用正则匹配即可。xxxxxx，是6位阿拉伯数字，即你处理这个案件的当日日期，比如是2026-04-16。截取，年份+月份+日，形成6位数字。

## 新需求二

1. 生成新的命令，python3 main.py --civil。--civil参数的具体处理逻辑：对于从财产线索生成协助执行通知书文件这个步骤，需要区分鹰眼还是外勤，鹰眼类的不需要生成协助执行通知书。此外支付宝的也不需要生成，财付通需要生成。
2. 对于其他命令，例如python3 main.py --new命令，可以叠加--civil参数。其他命令如python3 main.py -c/-f命令在叠加--civil参数时，需要按照上述第1条的--civil参数的处理逻辑，即鹰眼类不需要生成协助执行通知书，此外支付宝的也不需要生成，财付通需要生成。
3. python3 main.py，所有的财产线索都需要生成，即一条财产线索对应生成一份协助执行通知书。
4. 写一个重置逻辑的命令，python main.py -r/--reset。具体逻辑：就是清空outputs和original_files目录下的子目录及其文件，保留outputs目录。
